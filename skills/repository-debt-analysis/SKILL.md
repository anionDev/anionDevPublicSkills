---
name: repository-debt-analysis
description: Analyzes the repository to identify technical debt and areas for non-functional improvements.
---
Perform an analysis of the repository to ensure code quality and maintainability.
Things to consider:
- Identify potential code smells or anti-patterns.
- Evaluate the overall structure and organization of the code.
- Assess the readability and maintainability of the code.
- Search for any redundant or duplicate code that can be refactored.
- Search for technical debts.
In the end provide a summary of the findings and make suggestions for improvements and refactoring where necessary.
Do not actually change any file.
